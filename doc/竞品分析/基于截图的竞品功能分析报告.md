# 基于截图（p1–p4）的竞品功能分析报告

说明：以下分析基于目录 `doc/竞品分析/` 中的截图（p1.jpg、p2.jpg、p3.jpg、p4.jpg）的可见界面元素与常见交互范式进行归纳推断；对无法从静态截图确认的能力，标记为「待验证」。

## 一、整体定位与信息架构印象

- 导航层级：左侧主导航（项目/看板/进度/预警/报表等），上部工具条（搜索、导入、新建）。
- 页面类型：
  - 工作台/项目概览页（指标卡片 + 快捷筛选）；
  - 看板页（多列状态、任务卡片、列统计/图表）；
  - 时间轴/甘特（里程碑、依赖、计划/实际对比）；
  - 预警/消息页（风险列表、订阅/通知动作）。
- 视觉与布局：卡片式信息容器、留白明显；列表/卡片 + 右侧明确信息区；顶部统一的搜索/CTA 区域。

## 二、功能清单（按模块归类）

### 2.1 项目/任务基础

- 项目：创建/编辑/归档、负责人/成员（从顶部与卡片元素可见）、项目状态/标签（推断）。
- 任务：
  - 基础字段：标题、状态（未开始/进行中/阻塞/已完成）、负责人、进度百分比（截图标注“进度 xx%”）。
  - 交互：在列内展示为「任务卡片」；列表项右侧展示 Owner / 状态徽标。
  - 分组：按状态列（列标题如“未开始/进行中/阻塞/已完成”）。
  - 子任务/清单（待验证：截图未直观展示折叠图标）。
- 依赖/里程碑（待验证：时间轴页可能呈现）。

### 2.2 视图与可视化

- 看板视图：
  - 列管理：多列并排、列标题、列内卡片；列内卡片具备一致的卡片样式（圆角、阴影、标签）。
  - 统计：列上方或额外卡片区提供可视化（从另外截图推断有图表卡片，如饼/柱/折线）。
  - 拖拽（待验证：截图未显示拖拽手柄，常规看板默认支持）。
- 报表/图表：
  - 饼图：状态占比（未开始/进行中/阻塞/完成）。
  - 柱状图：各列任务数量/人均在制。
  - 折线图：近 7 日完成量/创建量趋势。
  - 燃尽/燃起（待验证）。
- 时间轴/甘特（从“进度/时间相关截图”推断）：
  - 计划 vs 实际、任务条/关键路径（待验证）。

### 2.3 搜索、筛选与导出

- 顶部全局搜索：占位提示“搜索项目/任务/成员…”。
- 条件筛选：
  - 负责人/状态/部门（从筛选控件样式与选项推断）。
  - 时间区间（待验证）。
- 导出能力：CSV/Excel（按钮“导出”/“导出 CSV”）。
- 保存视图（待验证）。

### 2.4 协作与通知

- 订阅/通知：
  - 动作按钮“通知”出现在预警/提醒项卡片（推断可对相关人发送提醒）。
- 评论/附件（待验证：截图未见评论输入区）。

### 2.5 预警与风险

- 预警列表：
  - 条目包含等级（高/中/信息）、说明文本、关联对象（例如“接口联调”）、「通知」动作。
  - 风险类型：进度滞后、依赖未完成、到期提醒等（从文案推断）。
- 预警生成逻辑（待验证）：规则/AI；支持订阅范围（负责人/成员/订阅者）。

### 2.6 权限与角色（截图无法完全确认，基于通用实践推断）

- 角色分层：管理员、项目负责人、成员、访客/只读。
- 访问控制：项目级/任务级可见性、导出权限（待验证）。

### 2.7 自动化与集成（待验证）

- IM 集成：企业微信/飞书 Webhook（从“通知”与中文企业场景推断）。
- 规则触发：状态变化/到期 → 通知/分配/改字段（从预警文案推断）。

## 三、基于截图的页面拆解

### 3.1 顶部工具区

- 组件：Logo + 标题、全局搜索、导入按钮、新建项目按钮（主 CTA）。
- 设计逻辑：
  - 将高频操作（搜索、新建）置于固定位置；
  - 导入入口靠近新建，降低从外部系统迁移成本。

### 3.2 侧边导航

- 分组：导航标题“导航”，下含“项目/看板/进度/预警”等入口。
- 设计逻辑：
  - 以任务推进主视图（看板/时间轴）为核心，左侧一级结构稳定呈现，移动端收缩到底部（自适应）。

### 3.3 看板列与任务卡片

- 列：未开始、进行中、阻塞、已完成。
- 卡片要素：标题、进度百分比、Owner、状态徽标；卡片圆角 + 轻阴影，作为信息单元。
- 设计逻辑：
  - 统一卡片信息骨架，保证浏览密度与层级清晰；
  - 进度与状态双重编码，兼顾概览与判断；
  - Owner 徽标固定位置，降低视线搜寻成本。

### 3.4 图表卡片（看板上方/独立区）

- 饼图：分布一眼览（状态/部门/优先级）。
- 柱状图：在制/各列数量对比，便于识别瓶颈列。
- 折线图：趋势（近 7 天或周维度），用于监控短期节奏。
- 设计逻辑：
  - “概览 → 下钻”路径：先看指标/图，再到看板列定位问题；
  - 多图并列，支持移动端折行，保证自适应布局。

### 3.5 预警列表

- 列表项：等级点（颜色）、文案、关联对象、通知按钮。
- 设计逻辑：
  - 用显著色彩区分等级（危险/警告/信息）；
  - 动作近身（卡片内即可通知），缩短处理路径。

## 四、背后的设计逻辑与取舍

1. 以任务流转为主线，最短路径直达“看板/进度”视图；
2. 先概览（图表/指标）→ 再下钻（列/卡片）→ 最后处理（评论/通知/分配）；
3. 用卡片化统一信息密度与视觉节奏，降低视觉负担；
4. 用轻量的统计（饼/柱/线）支撑“瓶颈定位/节奏判断”，避免上手门槛高的复杂报表。

## 五、与我们产品的差距与落地建议

- 已具备：
  - 看板四列、卡片信息骨架、图表三件套（饼/柱/折线）、预警列表、全局搜索、导出按钮。
- 待增强：
  - 拖拽排序与列间移动（需引入 DnD 库）；
  - 任务详情面板（右侧抽屉：字段/评论/附件）；
  - 依赖与里程碑在时间轴上呈现（关键路径高亮）；
  - 预警“因子解释 + 一键分派/延误豁免”；
  - 筛选与保存视图（共享视图/个人视图）。
